Coding the right way

  1. Getting started

    • Create a new folder per project

    • Create 3 new files

      • Or at least the ones you need.

      If you REALLY only need JS, you can put the script into the HTML if its short.

    • Link them up.

      • Don't copy and paste into the console.

  2. Keep your code readable

      3. New rules at Developers Institute

    • Indent your $#@%ing code!

    • Meaningful variable names

      • x, x2, myX vs beer1, beer2 vs i/j

    • Meaningful function names

  3. Structure your code

    • Function declaration

    • Create and initialize variables

    • Execution

  4. Make your life easy

    • Start Simple - Break the problem down

      • capitalize 1st letter in an array of words.

    • Get it working the upgrade it

      • Use hardcoded values, add user input later (hangman)

    • Start small

      • 2 lives, not 10

    • Don't put alerts, its annoying.

      • Put console.log AND say what you are logging. (if/else & key/values)

  5. Enter the matrix - How to get really good

    • Learn your IDE

    • Learn the commandline

    • Debugging

    • Learning something new

      • Methods and Functions

      Eg: MDN Arrays vs Strings..

      - Look it up.

      - See who it belongs to.

      - Understand the method signature.

      - Open the console and play! Otherwise you won't understand or remember.

  6. How to get really really really ridiculously good

    • Don't be afraid to make mistakes

    • Try things for yourself

      • Try 3 different solutions before asking someone else

    • Work things out with someone

    • Do the daily challenge

      • At least do a piece of it.

    • Build cool stuff, break cool stuff

      • And then rebuild it again.
